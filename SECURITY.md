# 🛡️ Security Policy - Synthesis-Core

Security is a top priority at Synthesis-Core Studio. Since we develop low-level systems, kernel-adjacent tools, and server architecture, we are committed to maintaining a secure environment for all our repositories.

---

## 💻 Supported Versions

Only the active versions listed below receive security updates and patches:

| Version | Supported |
| --- | --- |
| Latest Releases (Main branch) | ✅ Yes |
| Legacy / Older Versions | ❌ No |

---

## 🚨 Reporting a Vulnerability

If you find a security vulnerability or a networking exploit in our terminal emulator, Kotlin backend, or server infrastructure, **DO NOT open a public Issue**. 

Opening a public issue exposes the flaw to malicious actors before the Core Team can fix it. 

### 📬 How to report safely:
1. Send a private email detailing the exploit to: **Synthesis-Core.Studio@proton.me**
2. Include a brief description of the bug and the steps or code needed to reproduce it.
3. Vini (Security Director) and Samuel (CTO) will review the report privately and work on a security patch (`fix:`).

Thank you for helping us keep the Synthesis-Core ecosystem safe for our 7 members and future users!
