# 🛠️ Synthesis-Core Contributing Guidelines

Welcome to the Synthesis-Core Studio! If you are reading this, you are probably part of our school team or looking to help our studio develop high-performance systems. 

Please read these rules carefully before making your first commit.

---

## 👑 Board of Directors (The Funders)
All structural decisions, repository access, and final code reviews go through the 5 founders:
- **Samuel (lucsaml)**: Chief Technology Officer (CTO) & Low-Level Infrastructure.
- **Vinicius**: Networking & System Security.
- **Leandro**: UI/UX Design Architecture.
- **Valentina**: Art & Brand Identity.
- **Theo**: Planning & Release Strategy.

---

## 📜 Golden Rules for Developers
To keep our code organized and our servers alive, you must follow these rules:

- **No direct pushes to `main`**: Always create a new branch for your task (`git checkout -b feature/my-task`).
- **Ask the CTO first**: If you want to modify anything inside repositories' code or server routing, lucsaml must approve it beforehand.

---

## 💬 Code Commits Standard
We follow the **Conventional Commits** standard. Your commit messages must start with one of these tags:

- `feat:` When adding a brand new feature or system instruction.
- `fix:` When fixing a bug that breaks the code.
- `docs:` For updates in READMEs, guides, or comments (like this file!).
- `chore:` For project maintenance, updating dependencies, or fixing `.gitignore`.

Example: `git commit -m "docs: add core contributing rules"`

---

## 🚀 How to Submit Your Contribution
1. **Fork** the repository you want to help with.
2. Clone it to your machine and create a new branch.
3. Make your changes using the correct commit style.
4. Push to your fork and open a **Pull Request (PR)**.
5. Wait for the Board of Directors to review, test, and merge your code!

> [!TIP]
> Need help setting up your Git, Docker, or Arch Linux workspace? Ping lucsaml or tucano on Discord/Matrix!
